# Arquitectura de Despliegue (Deployments)

## Frontend
El frontend está construido con Next.js 14 y Tailwind CSS. El despliegue de producción se realiza en Vercel, y se utiliza GitHub Actions para correr las pruebas automatizadas (linting y tests unitarios) antes de cada paso a producción.

## Backend
El backend utiliza FastAPI (Python). Está empaquetado en un contenedor Docker. La imagen de Docker se almacena en Amazon ECR y se despliega en un clúster de Amazon ECS usando Fargate.

## Resolución de Errores Comunes de Despliegue
Si ocurre un error "502 Bad Gateway" después de un despliegue del backend, significa que el contenedor falló al iniciar. El equipo de soporte debe revisar los logs de CloudWatch. La causa más común es una variable de entorno faltante en AWS Secrets Manager.
